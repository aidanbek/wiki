Bloom Filters (Фильтры Блума)

Вероятностная структура данных для эффективной проверки членства элемента в множестве - space-efficient с допустимыми
false positives. Bit array размером m + k hash функций, добавление элемента устанавливает k битов в позициях hash_i(
element). Проверка членства - все k битов установлены значит "возможно в множестве", хоть один не установлен - "точно не
в множестве". False positive возможен (биты могли установить другие элементы), false negative невозможен - если элемент
добавлен, его биты гарантированно установлены. Probability false positive зависит от m, n (количество элементов), k -
оптимальное k ≈ (m/n)ln(2). Чрезвычайно компактны - для 1% false positive нужно ~10 bits per element независимо от
размера элементов. Нельзя удалять элементы (unless counting bloom filter), нельзя перечислить содержимое. Применения:
кэш проверки перед дорогим запросом (DB, disk), распределённые системы (CDN edge checking), spell checkers, Bitcoin
lightweight clients. Используй когда space критичен и false positives приемлемы.
