Write-Through

Запись синхронно в кэш и DB - каждый write идёт в оба места, успех только когда оба подтверждены. Гарантирует consistency - кэш всегда актуален с DB. Высокая latency записи (два sequential writes), но чтение быстрое (всегда in cache). Используется для критичных данных где consistency важнее скорости записи. При падении кэша данные safe в DB. Простая модель consistency для reasoning. Недостаток: write latency, пишем в кэш даже данные которые не будут читаться (cache pollution).
