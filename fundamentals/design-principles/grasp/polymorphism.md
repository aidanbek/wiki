Когда поведение зависит от типа, используй полиморфные операции вместо условных конструкций. Вместо if (paymentType == "card") ... else if (paymentType == "cash") создай интерфейс Payment с реализациями CardPayment, CashPayment. Устраняет type-checking, делает систему открытой для расширения (Open/Closed). Позволяет добавлять новые типы без изменения существующего кода.
