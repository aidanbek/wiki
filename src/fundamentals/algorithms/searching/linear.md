Сложность O(n)
Linear Search (Последовательный поиск)

Перебор элементов по порядку до нахождения целевого или конца коллекции - простейший и универсальный алгоритм. Сложность O(n) worst/average case, O(1) best case (элемент в начале). Не требует предварительной обработки или особой организации данных - работает на unsorted, unindexed данных. Единственный вариант для unsorted данных без дополнительных структур. Эффективен для маленьких коллекций (n < 10-20) где overhead более сложных алгоритмов не окупается. Может быть оптимизирован: sentinel для устранения проверки границ, move-to-front для улучшения locality, early exit если данные partially sorted.
