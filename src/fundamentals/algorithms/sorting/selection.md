Selection Sort (Сортировка выбором)

На каждой итерации находит минимальный элемент в неотсортированной части и помещает в конец отсортированной - простой концептуально алгоритм. Два указателя: граница отсортированной части и текущая позиция поиска минимума. O(n²) во всех случаях - даже для отсортированных данных выполняет полный поиск минимума. In-place (O(1) память), unstable (может менять порядок равных элементов при swap). Минимальное количество swap'ов - O(n), полезно когда запись дорога (flash memory, disk). Простота реализации делает его учебным примером, но практически бесполезен - insertion sort лучше для малых данных, quicksort/mergesort для больших.
