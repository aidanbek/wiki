Shell Sort (Сортировка Шелла)

Обобщение insertion sort с предварительным сортированием на больших промежутках (gap) - уменьшает инверсии перед
финальным проходом. Сортирует элементы отстоящие на gap, затем уменьшает gap (обычно gap=gap/2 или более хитрые
последовательности), финальный проход с gap=1 - обычный insertion sort. Сложность зависит от gap sequence: worst O(n²)
для некоторых, O(n^(3/2)) для Knuth's (3^k-1)/2, O(n log² n) для Sedgewick's. In-place, unstable, adaptive (лучше на
частично отсортированных). Практически быстрее insertion sort на средних данных (тысячи элементов), медленнее
quicksort/mergesort на больших. Простая реализация (insertion sort с gap), хорош для embedded systems где нельзя
использовать O(n log n) память или рекурсию. Исторический интерес, редко используется в production - quicksort
доминирует.
