Удаление записи с наименьшей частотой обращений - предположение что популярные данные останутся популярными. Отслеживание счётчика обращений для каждой записи, evict с минимальным count. Лучше LRU для skewed workloads (hot items стабильны). Проблема: старые записи накапливают high counts, новые популярные вытесняются. Window-LFU, TinyLFU решают через time windows или sketch. Сложнее реализация, больше метаданных. Используется когда workload характеризуется небольшим stable working set.
