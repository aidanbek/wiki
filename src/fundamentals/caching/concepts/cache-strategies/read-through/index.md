Read-Through

Кэш сам загружает данные из DB при miss - прозрачная загрузка, приложение работает только с кэшем. Cache library/system
имеет встроенную логику fetch from source. Приложение делает cache.get(key) - кэш внутренне проверяет, при miss
загружает из DB, сохраняет, возвращает. Упрощает код приложения (нет fallback логики), централизует loading в кэш-слой.
Требует настройки data loader в кэш-системе. Холодный кэш медленнее - первый miss вызывает synchronous load.
