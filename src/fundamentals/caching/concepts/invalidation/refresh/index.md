Refresh (Background Refresh)

Асинхронное обновление данных в кэше до истечения TTL - proactive вместо reactive. Worker процесс периодически обновляет
популярные записи из источника. Пользователи всегда получают cached данные без задержки на fetch. Для критичных данных
где hit всегда желателен - главная страница, топ товаров, агрегированная статистика. Требует мониторинга популярности (
что refresh'ить) и координации (не дублировать refresh). Может комбинироваться с SWR для fallback.
