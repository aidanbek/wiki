TTL (Time To Live)

Время жизни кэшированных данных после которого они считаются stale и удаляются/revalidate - простейшая стратегия.
Устанавливается при записи в кэш: set(key, value, ttl=300) - через 5 минут запись испаряется. Cache-Control: max-age в
HTTP, EXPIRE в Redis, expiration time в Memcached. Пассивная invalidation - данные не удаляются сразу по истечении TTL,
а при следующем обращении или фоновом процессе. Выбор TTL зависит от частоты изменения данных: статика (дни/месяцы), API
ответы (минуты/часы), session data (часы).
