Разделение на три слоя: Model (бизнес-логика и данные), View (представление), Controller (обработка ввода и координация). Controller принимает запрос, обновляет Model, выбирает View для отображения. Model уведомляет View об изменениях через Observer. Классический паттерн для UI-приложений, изолирует бизнес-логику от представления. Многочисленные вариации: MVP, MVVM, каждая корректирует направление зависимостей и роль компонентов.
