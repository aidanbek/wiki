Принцип, при котором фреймворк/контейнер управляет потоком выполнения и созданием объектов вместо прикладного кода. "
Don't call us, we'll call you" - Hollywood Principle. Вместо main() создающего все объекты, контейнер создаёт и
связывает их по конфигурации. Обобщающий термин для DI, Service Locator, Template Method, Factory. Инверсия - не твой
код вызывает библиотеку, а библиотека вызывает твой код через callbacks/hooks.
