DDD (Domain-Driven Design)

Подход к разработке сложных систем через глубокое понимание domain и моделирование бизнес-логики - Eric Evans's
methodology. Ubiquitous Language - единый язык между разработчиками и domain experts, код отражает язык бизнеса.
Strategic patterns (bounded contexts, context mapping) для организации больших систем. Tactical patterns (entities,
value objects, aggregates, domain events) для моделирования внутри контекста. Фокус на core domain, separation of
concerns, rich domain model вместо anemic. Борется с complexity через linguistic clarity и explicit modeling. Overhead
оправдан только для complex domains, overkill для CRUD apps.
